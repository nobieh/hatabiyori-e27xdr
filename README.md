# 畑日和 v2（独立版）

Claude・ChatGPTどちらにも依存しない、単体で動く畑日和です。
GitHub Pages（無料）に置けば、お父さまはURLを開いてホーム画面に追加するだけ。
誰の課金もいりません。記録はお父さまのスマホの中に保存されます。

## ファイル一覧（この5つで全部）

| ファイル | 役割 |
|---|---|
| `index.html` | アプリ本体（画面・ロジック・作物データすべて入り） |
| `manifest.json` | 「ホーム画面に追加」でアプリらしく開くための設定 |
| `sw.js` | 電波が弱くても開けるようにするオフライン対応 |
| `icon-192.png` / `icon-512.png` | ホーム画面のアイコン |

## 公開のしかた（GitHub Pages・ブラウザ操作だけ）

1. [github.com](https://github.com) にログイン（なければ無料登録）
2. 右上の「＋」→ **New repository** → 名前を `hatabiyori` に、**Public** のまま **Create repository**
3. 「**uploading an existing file**」のリンクを押し、上の5ファイルをまとめてドラッグ＆ドロップ → **Commit changes**
4. リポジトリの **Settings → Pages** → Branch を `main`／`/(root)` にして **Save**
5. 1〜2分待つと `https://ユーザー名.github.io/hatabiyori/` が本番URLになります

### お父さまのスマホでの初期設定

1. 本番URLをLINEなどで送って開いてもらう
2. ブラウザのメニュー（⋮）→「**ホーム画面に追加**」
3. 以後はホーム画面の🌱アイコンから起動（見た目は普通のアプリです）

## ふだんの運用で大事なこと

- **月に一度のバックアップ**：「せってい」→「📤 記録を送る」→ LINEでご家族へ。30日空くとアプリが黄色い帯でやさしく催促します。届いたJSONファイルは保管してください。
- **AIそうだん**：「きょう」画面の「🤖 AIにそうだん」→ コピー → ChatGPTに貼り付け。作物・日数・今日の天気・予定・収穫・メモ入りの要約が渡るので、ChatGPTが畑を分かったうえで答えます。
  - 発展形：お父さまのChatGPTに「畑の相談役」カスタムGPTを作り、この要約＋バックアップJSONを渡す運用にすると、さらに気が利きます（第二弾として構築可能）。

## 更新のしかた（開発サイクル）

1. `index.html` をClaude Code（またはChatGPT）で編集
2. 更新の目印として `HB.VERSION`（index.html内）と `VERSION`（sw.js内）の数字を上げる
3. GitHubのリポジトリで対象ファイルを開き、鉛筆アイコン → 貼り替え → Commit
4. お父さま側は、アプリを開き直せば新版が届きます（本体はネット優先で読む設計）

## 旧・畑日和（Claude版）からの引っ越し

1. 旧アプリの**バックアップ書き出し**で、いまの記録JSONを取り出す
2. そのJSON（と可能なら旧 `hatake-note.jsx`）をClaudeに渡す
3. 変換した新形式JSONを受け取り、新アプリの「せってい」→「📥 バックアップから戻す」で読み込む
4. 旧版の作物データベースは移植済み（v2.1.0）。残るのは記録データの変換のみ

※新アプリは旧形式のJSONを読むと「引っ越し用に変換できます」と案内し、データを壊さず弾きます。

## いまの制限・注意

- **作物データは本家・畑日和v1から移植済み**（v1の34種＝野菜27種＋バラ7種を忠実に移植し、新版で追加した5種を加えた全39種。v2.1.0〜）。
- 記録は**このスマホの中だけ**（localStorage）。ブラウザの「閲覧データ削除」で消えるため、バックアップ習慣が命綱です。容量の目安は約5MB（写真は640pxに自動縮小、いっぱいになると赤帯でお知らせ）。
- 天気は Open-Meteo（無料・APIキー不要）。地点は `index.html` 冒頭の `HB.LOC` で変更できます（現在：因島）。
- AI相談機能そのものはアプリから外しました（Claude専用APIのため）。役割はChatGPT側に引っ越しです。

## Drive自動保存（v2.3で統合）

- 場所は「せってい → Drive自動保存 ☁️」。写真を除く全記録＋今日の天気＋作業状況を、お父さまのGoogle Driveの「畑日和」フォルダへ自動同期します（アプリを開いた時と収穫記録のあと、多くても6時間に1回）。
- Drive側には `hatabiyori_summary.txt`（GPTがそのまま読める要約）、`hatabiyori_harvests.csv`（収穫の追記ログ）、`hatabiyori_daily_log.csv`（天気×作業の日次ログ）、`hatabiyori_latest.json`（全体スナップショット）ができます。
- 初期設定（約10分・一度だけ）は別添の `DRIVE_SETUP.md` と `hatabiyori_gas_receiver.gs` を使用。**この2ファイルは公開リポジトリに入れないこと**（合言葉を含むため）。設定するまで同期はオフのまま安全に動きます。
- 同期が30日以内に成功していれば、LINEバックアップの催促バナーは出ません（手動バックアップは保険として残ります）。

### 役わり・設定リンク・復元（v2.4で統合。Driveが親玉、端末は窓）

- **端末の役わり**（うら設定）: 「本番（送る）」＝お父さまのスマホ、「見るだけ（受け取る）」＝家族のPC等。見るだけ端末は開くたびDriveの最新を映すミラーになり、送信は一切しません。
- **🔗 設定リンク**: うら設定からURL＋あいことば入りのリンクを発行。LINEで送って開くだけで設定完了（手入力不要）。リンクは家族外に出さないこと。
- **⬇️ Driveから読み込む**: 復元・機種変・ミラー更新用。この端末の写真は残したまま、中身をDriveの最新に置きかえます。
- 要件: 受付係を **v1.1**（読み出し対応）以上に更新すること（「デプロイを管理→編集→新バージョン」でURLは変わりません）。

### 📆 カレンダー連携（v2.5）

- 受付係 **v1.3** がGoogleカレンダーから農業の予定だけを抽出し、アプリの「きょう」「こよみ」「AI要約」に📆付きで合流させます（完了チェック可・未完の過去分はやり残しに合流）。
- 拾う条件: ①「畑日和」という名前のカレンダーは**全予定** ②メインカレンダーは**【畑】**か**農業キーワード**（受付係の `CAL_KEYWORDS` で自由に追加）。窓は前7日〜先180日。
- 📆予定の直し方は二層: そのままなら**Googleカレンダー側で編集**(次にアプリを開いた時に反映)。アプリで直したければ、📆行をタップ→**「✏️ アプリに写して編集」**で自分の予定に変換(以後、日付・内容・メモを自由に編集/削除。元の📆は自動で隠れて二重にならない)。うら設定でオフにもできます。
- **v2.6**: 作物にひもづかない「📝 じぶんの予定」を新設（`state.tasks`・Drive同期対象）。「こよみ」の日付詳細に**「＋ この日に予定を追加」**。既存のper-crop custom（作物詳細の「じぶんの予定を足す」）とは別物として共存。
- 要件: 受付係v1.3への更新時、**カレンダー閲覧の再承認**が一度だけ必要（例の赤い画面）。

## 開発メモ（Claude Code向け）

- `index.html` は4層構成：`<style>`（デザイントークン）→ `#hb-core`/`#hb-core2`（純ロジック：日付・保存・作物DB・予定エンジン・要約・バックアップ・天気・写真圧縮。UIなしで単体テスト可能）→ `text/babel` ×4（React UI）→ 起動・SW登録。
- React 18 UMD + babel-standalone **7.26.4**（cdnjs固定）。classicランタイム（`React.createElement`）前提。バージョンを上げる場合は automatic runtime にならないか要確認。
- 検証済み：コアロジック単体テスト（日付計算・予定生成・バラ年間予定・収穫窓・要約・バックアップ往復・旧形式判定）、全JSXブロックの構文、jsdomでの実描画（挨拶・タブ・今日の作業・圏外時の天気・自動保存）。
